# M2 - PDF Manipulation Engine

## Objective

Implement the PDF manipulation backend and tests behind Application
abstractions.

## Features Implemented

- PDF inspection.
- Page geometry reading.
- Merge.
- Split by page ranges.
- Extract pages.
- Rotate pages.
- Metadata update.
- Encrypted PDF detection when a password is provided.
- Corrupt PDF and invalid password error mapping.
- Atomic output writes for single-output operations.
- Progress reporting and cancellation.
- Split output name templating.

## Architecture Summary

M2 introduced `IPdfEngine` in Application and implemented it with
`PdfSharpEngine` in Infrastructure. UI remains independent of PDFsharp.

`ComposeAsync`, `EncryptAsync`, `DecryptAsync` and `CompressAsync` remain
explicitly unsupported until later milestones.

- Added operation request records:

  `CompressRequest` and `UpdateMetadataRequest`.
- Added `OperationProgress`.

## Test Summary

Current Infrastructure suite has 31 passing tests. PDF engine tests use real
PDF files generated by PDFsharp and assert page identity through page geometry.

## Manual Verification

- Run `dotnet test --no-restore`.
- Exercise Merge through the UI to confirm the engine is called by a
  user-facing workflow.
- For lower-level split/extract/rotate/metadata behavior, use tests until UI
  pages are implemented.

## Known Issues

- Split, Extract and Rotate are engine capabilities but do not yet have
  dedicated UI pages.
- Metadata update exists in the engine but the Metadata editor is not
  implemented.
- Compose, Compress and Protect operations are not implemented.
- Password handling is not exposed in the Merge UI.

## Lessons Learned

- PDFsharp gives enough manipulation support for the early toolset.
- Real file integration tests are valuable for PDF operations.
- Atomic writes are essential for failed or cancelled PDF operations.

## Developer Handover

### Current Implementation

`PdfSharpEngine` is registered as a singleton `IPdfEngine`.

### Completed Work

The manipulation engine is ready for UI workflows that need inspect, merge,
split, extract, rotate and metadata update.

### Remaining Work

Implement missing UI pages and unsupported engine operations.

### Recommended Next Milestone

Continue connecting implemented engine operations to user-facing workflows, or
implement rendering if visual organization is next.

### Architectural Warnings

Keep PDFsharp isolated to Infrastructure. Do not reference PDFsharp from UI.

### Files Requiring Attention

files; keep tests around error mapping.
