# M1 - Application Shell

## Objective

Establish the application foundation: solution structure, architecture,
desktop shell, design system, settings, logging and tests.

## Features Implemented

- .NET 10 solution and centralized build configuration.
- Avalonia desktop application shell.
- Home page and tool catalog.
- Sidebar navigation.
- Keyboard shortcuts for primary navigation.
- Settings page with theme selection.
- Light, Dark and System theme support.
- Persisted window size and maximized state.
- JSON settings persistence.
- Recent files service.
- Rolling file logger.
- Null update service.
- Baseline test projects.

## Architecture Summary

M1 established the Clean Architecture layout:

- Domain for PDF value types and rules.
- Application for abstractions and models.
- Infrastructure for concrete persistence, logging and update services.
- UI for Avalonia views, view models and theme resources.
- Desktop as the composition root.
- SDK as the future plugin contract assembly.

- Run the app with `dotnet run --project src/Acroball.Desktop`.
- Confirm the shell opens.

- Switch between System, Light and Dark themes.
- Close and reopen to confirm window state persistence.

## Known Issues

- Some source file names still contain old `Quire` naming.
- User-facing tool pages other than Merge are placeholders.
- No rendered UI smoke test harness is present.

## Lessons Learned

- Keeping ViewLocator explicit protects navigation from trimming and reflection
  assumptions.
- Centralized theme resources make light/dark support straightforward.
- Settings and logs are small enough to own without additional dependencies.
